Data Mining · Dia 1

Roteiro 1 — Análise e Processamento de Dados

Base: Dados/tramontina_clientes.csv
Referência: Notebooks/01_Analise_e_Processamento_de_Dados.ipynb

0. Preparação

  1. Abra o Orange Data Mining.
  2. Arraste o widget File para o canvas.
  3. No File, carregue Dados/tramontina_clientes.csv.
  4. Em Columns, ajuste os papéis: ID_Cliente → meta; Churn → target; demais → feature.

1. Exploração (ramo direto do File)

Conecte todos os widgets desta seção diretamente à saída Data do File.

1.1Data Table

  1. Adicione Data Table · conecte File → Data Table.
  2. Confira as colunas e o nº de linhas.

1.2Column Statistics

  1. Adicione Column Statistics · conecte File → Column Statistics.
  2. Observe média, mediana, mínimo, máximo e dispersão.

1.3Distributions

  1. Adicione Distributions · conecte File → Distributions.
  2. Variable: Faturamento_Anual_R$ · Split by: Churn · Bins: 30.

1.4Box Plot

  1. Adicione Box Plot · conecte File → Box Plot.
  2. Variable: Faturamento_Anual_R$ · Subgroups: Canal_Venda.
  3. Anote o limite superior (Q3 + 1,5·IQR) — usado no Select Rows.

1.5Scatter Plot

  1. Adicione Scatter Plot · conecte File → Scatter Plot.
  2. Axis x: Ticket_Medio_R$ · Axis y: Faturamento_Anual_R$ · Color: Churn.

1.6Correlations

  1. Adicione Correlations · conecte File → Correlations.
  2. Método Pearson; ordene pelo valor absoluto.

1.7Formula

  1. Adicione Formula (Feature Constructor) · conecte File → Formula.
  2. Crie os atributos:
NomeTipoExpressão
Faturamento_por_PedidoNumericFaturamento_Anual_R$ / Frequencia_pedidos_ano if Frequencia_pedidos_ano > 0 else 0
Total_Categorias_CompradasNumericComprou_Cutelaria + Comprou_Panelas + Comprou_Utilidades_Cozinha + Comprou_Ferramentas + Comprou_Eletroportateis + Comprou_Moveis + Comprou_Talheres + Comprou_Jardinagem
Cliente_InsatisfeitoCategorical1 if Satisfacao_NPS_1a10 <= 6 else 0
Porte_Cliente_ordNumeric1 if Porte_Cliente == "Pequeno" else 2 if Porte_Cliente == "Médio" else 3
Selecione cada variável pelo seletor do widget para inserir o nome correto (colunas com $ e % são referenciadas pelo seletor).

2. Processamento (pipeline)

2.1Impute

  1. Adicione Impute · conecte File → Impute.
  2. Default method: Average/Most frequent.
  3. Cobertura para Desconto_Medio_% e Satisfacao_NPS_1a10.

2.2Select Rows

  1. Adicione Select Rows · conecte Impute → Select Rows.
  2. Condição: Faturamento_Anual_R$ · is below · [limite superior anotado no Box Plot].
  3. Send automatically ligado.

2.3Preprocess

  1. Adicione Preprocess · conecte Select Rows → Preprocess.
  2. Continuize Discrete Variables → One attribute per value (one-hot: Regiao, Canal_Venda, Segmento).
  3. Normalize Features → Standardize to μ=0, σ²=1.

2.4Data Table (1)

  1. Adicione Data Table · conecte Preprocess → Data Table (1).
  2. Confira as colunas one-hot e os valores padronizados.

2.5Formula (1)

  1. Adicione Formula · conecte Preprocess → Formula (1).
  2. Atributos derivados sobre os dados processados:
NomeTipoExpressão
Faturamento_por_PedidoNumericFaturamento_Anual_R$ / Frequencia_pedidos_ano if Frequencia_pedidos_ano > 0 else 0
Total_Categorias_CompradasNumericComprou_Cutelaria + ... + Comprou_Jardinagem

2.6Data Table (2)

  1. Adicione Data Table · conecte Formula (1) → Data Table (2).

2.7Save Data (exportação)

  1. Adicione Save Data · conecte Formula (1) → Save Data.
  2. Salve como tramontina_clientes_processado.csv.

Machine Learning · Dia 2

Roteiro 2 — Treinamento de Modelos

Bases: Dados/tramontina_clientes.csv (classificação) · Dados/tramontina_clustering.csv (clustering)
Referência: Notebooks/02_Treinamento_de_Modelos.ipynb

Parte A — Classificação de Churn

A.0Carregamento

  1. Adicione File · carregue Dados/tramontina_clientes.csv.
  2. Em Columns: ID_Cliente → meta; Churn → target; demais → feature.

A.1Impute

  1. Adicione Impute · conecte File → Impute.
  2. Default method: Average/Most frequent; cobre Desconto_Medio_% e Satisfacao_NPS_1a10.

A.2Preprocess

  1. Adicione Preprocess · conecte Impute → Preprocess.
  2. Continuize Discrete Variables → One attribute per value (Regiao, Canal_Venda, Segmento).
  3. Normalize Features → Standardize to μ=0, σ²=1.

A.3Modelos (learners)

Adicione os quatro widgets de aprendizado (serão ligados ao Test and Score):

WidgetParâmetro
kNNNumber of neighbors = 5
TreeLimit the maximal tree depth = 5
Logistic RegressionRegularization = Ridge (L2)
Random ForestNumber of trees = 100

A.4Test and Score

  1. Adicione Test and Score.
  2. Conecte Preprocess → Test and Score (Data).
  3. Conecte cada learner: kNN → Test and Score, Tree → Test and Score, Logistic Regression → Test and Score, Random Forest → Test and Score.
  4. Sampling: Cross validation · Folds = 5 · Stratified.
  5. Target class: Sim.
  6. Compare AUC, CA, F1, Precision, Recall.

A.5Matriz de confusão

  1. Adicione Confusion Matrix · conecte Test and Score → Confusion Matrix.
  2. Selecione um modelo e observe VP / FP / VN / FN.

A.6Curva ROC

  1. Adicione ROC Analysis · conecte Test and Score → ROC Analysis.
  2. Target: Sim; compare as curvas dos quatro modelos.

A.7Ranking de atributos (opcional)

  1. Adicione Rank · conecte Preprocess → Rank.
  2. Ordene por Gain ratio ou Gini.

Parte B — Segmentação com k-Means

B.0Carregamento

  1. Adicione um novo File · carregue Dados/tramontina_clustering.csv.
  2. Em Columns: ID_Cliente → meta; Canal_Predominante → meta; demais → feature.

B.1Select Columns

  1. Adicione Select Columns · conecte File → Select Columns.
  2. Mantenha como features: Gasto_Mensal_R$, Itens_por_Pedido, Frequencia_Compras_mes, Perc_Linha_Cozinha_Profissional, Anos_como_Cliente.

B.2Preprocess

  1. Adicione Preprocess · conecte Select Columns → Preprocess.
  2. Normalize Features → Standardize to μ=0, σ²=1.

B.3k-Means

  1. Adicione k-Means · conecte Preprocess → k-Means.
  2. Number of clusters: Fixed = 3.
  3. Para comparar, marque From 2 to 8 e observe a coluna Silhouette.
  4. Initialization: k-Means++ · Re-runs: 10.

B.4Silhouette Plot

  1. Adicione Silhouette Plot · conecte k-Means → Silhouette Plot.
  2. Cluster variable: Cluster.

B.5PCA + Scatter Plot

  1. Adicione PCA · conecte k-Means → PCA · Components: 2.
  2. Adicione Scatter Plot · conecte PCA → Scatter Plot.
  3. Axis x: PC1 · Axis y: PC2 · Color: Cluster.

B.6Perfil dos clusters

  1. Adicione Box Plot · conecte k-Means → Box Plot · Subgroups: Cluster.
  2. Percorra as cinco variáveis para perfilar cada grupo.
  3. (Opcional) Data Table ligada a k-Means para inspecionar o Cluster por cliente.
